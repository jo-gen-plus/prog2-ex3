迷路クラスの設計
プレイヤーの初期位置（スタート地点），迷路サイズ，迷路情報の保存について
    String filename = "./src/main/jp/ac/uryukyu/ie/e165715/map.txt";
            List<String> strings = new ArrayList<>(); // ファイルから読み込んだ文字列を保存するためのリスト。
            Scanner scanner;
    ファイルから文字列をリストとして保存するプログラムを使用し，保存したリストから必要な情報を引き出すようにする。

map.txtと同形式のテキストに対する互換性をもつようにするには
    上記のようにファイルの文字列をリストとして保存し，リストから情報を引き出す際に，何行目に必要な情報が有るかという形式で引き出してしまえば解決すると考える。

アクセサ(getter/setter)の必要性について
    安全性と直しやすさを考えてフィールド変数はカプセル化してしまうので，引き出した情報を利用してプログラムを作る際に必要。
迷路クラスが果たすべき機能について
    map.txtと同じ形式のファイルから行ごとに情報を取り出し，変数に保存，アクセサを用意して外部からアクセス出来るようにするのが役目


 プレイヤークラスの設計
プレイヤーを初期位置から１マス動かすにはプレイヤーが現在いる座標の値のどちらかの値を+1or-1する。その変更した座標がプレイヤーの現在位置となる。
その時、変更した座標の文字の存在を確かめ、存在するなら現在いる座標に数値を戻し処理をやり直す。存在しないならその座標に存在する空文字を”●”などのプレイヤーを表す記号に書き換える。
歩数は関数を作り、座標が変更されるごとに+1を行うようにする。ゴール（G）の座標にプレイヤーの座標が移った場合、”ゴール”とプリントして処理を終了する。
おそらくアクセサは必要である。
